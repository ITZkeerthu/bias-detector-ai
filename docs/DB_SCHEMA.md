## Overview

All data is stored in **Supabase** (managed PostgreSQL). The Node.js backend communicates with Supabase exclusively via the `@supabase/supabase-js` client using the `service_role` key (bypasses Row Level Security on the server side).

The Python `bias_detector_tool` does **not** write to the database directly — it returns analysis results to Node.js, which persists everything.

---

## Tables

### `users`

Stores extended user profile data. The primary authentication record lives in `auth.users` (managed by Supabase Auth); this table stores application-level metadata linked via the same UUID.

```sql
CREATE TABLE IF NOT EXISTS users (
    id            UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
    email         TEXT UNIQUE NOT NULL,
    role          TEXT NOT NULL DEFAULT 'analyst'
                      CHECK (role IN ('admin', 'analyst', 'viewer')),
    is_active     BOOLEAN NOT NULL DEFAULT TRUE,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

> **Note:** Passwords are managed entirely by Supabase Auth — no password hash is stored in this table. Login/signup goes through Supabase Auth endpoints, not a custom password hash comparison.

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID | FK → `auth.users.id`; matches the JWT `sub` claim |
| `email` | TEXT | Mirrors `auth.users.email` |
| `role` | TEXT | Application role for RBAC |
| `is_active` | BOOLEAN | Soft disable without deleting auth record |

### `sessions`

Each pipeline run (user message → full pipeline → response) is one session.

```sql
CREATE TABLE IF NOT EXISTS sessions (
    id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id           UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    model_config_id   TEXT NOT NULL,
    input_text        TEXT NOT NULL,
    final_response    TEXT NOT NULL,
    wrapper_triggered BOOLEAN NOT NULL DEFAULT FALSE,
    created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID | Session identifier |
| `user_id` | UUID | FK → `users.id` |
| `model_config_id` | TEXT | Matches a record in `model_configs.id` |
| `input_text` | TEXT | Raw user prompt |
| `final_response` | TEXT | Final AI response (post-wrapper if applied) |
| `wrapper_triggered` | BOOLEAN | Whether Phase 3 (Wrapper Engine) was activated |

---

### `bias_reports`

One bias report per session — stores the full pipeline analysis result.

```sql
CREATE TABLE IF NOT EXISTS bias_reports (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id          UUID UNIQUE NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    input_bias_score    FLOAT NOT NULL CHECK (input_bias_score BETWEEN 0 AND 100),
    output_bias_score   FLOAT CHECK (output_bias_score BETWEEN 0 AND 100),
    confidence_level    TEXT CHECK (confidence_level IN ('LOW', 'MEDIUM', 'HIGH', 'UNKNOWN')),
    wrapper_reasoning   TEXT,
    original_response   TEXT,
    wrapper_prompt      TEXT,
    created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID | Report identifier |
| `session_id` | UUID | FK → `sessions.id` (1:1) |
| `input_bias_score` | FLOAT | 0–100; from Phase 1 (Python bias detector) |
| `output_bias_score` | FLOAT | 0–100; from Phase 2b (Output Auditor LLM) |
| `confidence_level` | TEXT | Derived from `output_bias_score` |
| `wrapper_reasoning` | TEXT | Why the wrapper was triggered (from auditor LLM) |
| `original_response` | TEXT | Pre-wrapper LLM response (stored for audit trail) |
| `wrapper_prompt` | TEXT | Meta-prompt generated by Phase 3 |

### `protected_attribute_findings`

Each detected protected attribute from Phase 1 (Input Guard) is stored as a separate row.

```sql
CREATE TABLE IF NOT EXISTS protected_attribute_findings (
    id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    bias_report_id    UUID NOT NULL REFERENCES bias_reports(id) ON DELETE CASCADE,
    attribute         TEXT NOT NULL,
    confidence        FLOAT NOT NULL CHECK (confidence BETWEEN 0 AND 1),
    matched_text      TEXT,
    detection_method  TEXT NOT NULL,
    created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

| Column | Type | Notes |
|--------|------|-------|
| `attribute` | TEXT | e.g., `gender`, `race`, `age`, `national_origin` |
| `confidence` | FLOAT | 0–1; detection confidence from the Python model |
| `matched_text` | TEXT | The exact text that triggered detection |
| `detection_method` | TEXT | e.g., `name_origin_classifier`, `ner_entity`, `rule_based` |

### `proxy_variable_findings`

Each detected proxy variable from Phase 1 (Input Guard).

```sql
CREATE TABLE IF NOT EXISTS proxy_variable_findings (
    id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    bias_report_id    UUID NOT NULL REFERENCES bias_reports(id) ON DELETE CASCADE,
    variable          TEXT NOT NULL,
    mapped_to         TEXT NOT NULL,
    confidence        FLOAT NOT NULL CHECK (confidence BETWEEN 0 AND 1),
    created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

| Column | Type | Notes |
|--------|------|-------|
| `variable` | TEXT | e.g., `zip_code`, `university_name`, `employment_gap` |
| `mapped_to` | TEXT | The protected attribute this proxies (e.g., `race/socioeconomic`) |
| `confidence` | FLOAT | 0–1 |

### `log_entries`

Denormalized audit log for fast dashboard queries and CSV/JSONL export. Written in parallel with the structured `sessions` + `bias_reports` insert.

```sql
CREATE TABLE IF NOT EXISTS log_entries (
    id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id            UUID NOT NULL REFERENCES sessions(id) ON DELETE CASCADE,
    user_id               UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    prompt_preview        TEXT,
    input_bias_score      FLOAT,
    output_bias_score     FLOAT,
    confidence_level      TEXT,
    wrapper_triggered     BOOLEAN NOT NULL DEFAULT FALSE,
    protected_attributes  TEXT[],
    proxy_variables       TEXT[],
    model_id              TEXT,
    latency_ms            INTEGER,
    created_at            TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

| Column | Type | Notes |
|--------|------|-------|
| `prompt_preview` | TEXT | First 200 chars of `input_text` (for log table display) |
| `protected_attributes` | TEXT[] | Array of attribute names (e.g., `{gender, race}`) |
| `proxy_variables` | TEXT[] | Array of variable names (e.g., `{zip_code, university_name}`) |
| `latency_ms` | INTEGER | Total pipeline latency in milliseconds |

### `model_configs`

LLM model configurations. API keys are stored encrypted.

```sql
CREATE TABLE IF NOT EXISTS model_configs (
    id                  TEXT PRIMARY KEY,
    display_name        TEXT NOT NULL,
    provider            TEXT NOT NULL CHECK (provider IN ('openai', 'anthropic', 'gemini', 'mistral')),
    encrypted_api_key   TEXT NOT NULL,
    iv                  TEXT NOT NULL,
    auth_tag            TEXT NOT NULL,
    context_window      INTEGER,
    cost_per_1k_tokens  FLOAT,
    is_active           BOOLEAN NOT NULL DEFAULT TRUE,
    created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

| Column | Type | Notes |
|--------|------|-------|
| `id` | TEXT | Model slug (e.g., `gpt-4o`, `claude-3-5-sonnet`) |
| `provider` | TEXT | LLM provider name |
| `encrypted_api_key` | TEXT | AES-256-GCM encrypted; Base64 encoded |
| `iv` | TEXT | Initialization vector for AES-GCM; Base64 encoded |
| `auth_tag` | TEXT | GCM authentication tag; Base64 encoded |
| `context_window` | INTEGER | Max tokens for this model |
| `cost_per_1k_tokens` | FLOAT | USD cost per 1,000 tokens (for usage reporting) |

## Indexes

```sql
-- Fast lookup by user
CREATE INDEX idx_sessions_user_id ON sessions(user_id);
CREATE INDEX idx_sessions_created_at ON sessions(created_at DESC);

-- Fast bias report lookup
CREATE INDEX idx_bias_reports_session_id ON bias_reports(session_id);

-- Fast attribute/proxy lookup
CREATE INDEX idx_protected_attr_report ON protected_attribute_findings(bias_report_id);
CREATE INDEX idx_proxy_var_report ON proxy_variable_findings(bias_report_id);

-- Dashboard log queries
CREATE INDEX idx_log_entries_user_id ON log_entries(user_id);
CREATE INDEX idx_log_entries_created_at ON log_entries(created_at DESC);
CREATE INDEX idx_log_entries_wrapper ON log_entries(wrapper_triggered);
CREATE INDEX idx_log_entries_confidence ON log_entries(confidence_level);

-- Full-text search on audit log
CREATE INDEX idx_log_entries_prompt_fts
    ON log_entries USING GIN (to_tsvector(''english'', coalesce(prompt_preview, '''')));
```

## Supabase Client Usage

The backend uses `@supabase/supabase-js` with the `service_role` key for all server-side operations. Below are common patterns.

### Insert a pipeline result

```js
const { data: session, error } = await supabase
  .from('sessions')
  .insert({
    user_id: req.user.id,
    model_config_id: modelId,
    input_text: prompt,
    final_response: finalResponse,
    wrapper_triggered: wrapperApplied,
  })
  .select()
  .single();

const { data: report } = await supabase
  .from('bias_reports')
  .insert({
    session_id: session.id,
    input_bias_score: pipelineResult.inputBiasScore,
    output_bias_score: pipelineResult.outputBiasScore,
    confidence_level: pipelineResult.confidenceLevel,
    wrapper_reasoning: pipelineResult.wrapperReasoning,
    original_response: pipelineResult.originalResponse,
  })
  .select()
  .single();
```

### Query audit logs with filters

```js
let query = supabase
  .from('log_entries')
  .select('*', { count: 'exact' })
  .order('created_at', { ascending: false })
  .range(from, to);

if (wrapperOnly) query = query.eq('wrapper_triggered', true);
if (confidence)  query = query.eq('confidence_level', confidence);
if (startDate)   query = query.gte('created_at', startDate);
if (search)      query = query.textSearch('prompt_preview', search);

const { data, count, error } = await query;
```

### Fetch dashboard stats

```js
const { data: stats } = await supabase.rpc('get_dashboard_stats');
// Or using multiple queries:
const { count: totalSessions } = await supabase
  .from('sessions')
  .select('*', { count: 'exact', head: true });
```

## Supabase Auth Integration

Supabase Auth manages the full authentication lifecycle. The Node.js backend does **not** implement password hashing, token signing, or session management — all of that is delegated to Supabase.

**Auth flow:**

```
1. Frontend → POST /api/auth/login { email, password }
2. Backend  → supabase.auth.signInWithPassword({ email, password })
3. Supabase → Returns { session: { access_token, refresh_token, expires_in } }
4. Backend  → Returns access_token to frontend
5. Frontend → Stores access_token as localStorage["sb-access-token"]
6. Frontend → Sends: Authorization: Bearer <access_token> on all subsequent requests
7. Backend  → supabase.auth.getUser(token) to verify each request
```

**Linking `auth.users` to `users` table:**

When a new user is created in Supabase Auth, a corresponding row is inserted into the public `users` table via a Supabase database trigger or the backend signup handler:

```sql
-- Trigger to auto-create user profile on auth signup
CREATE OR REPLACE FUNCTION handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.users (id, email)
  VALUES (NEW.id, NEW.email);
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION handle_new_user();
```

**Token claims used by the backend:**

| JWT Claim | Usage |
|-----------|-------|
| `sub` | User UUID — matched to `users.id` |
| `email` | Display / logging purposes |
| `role` | Supabase default role (not the same as `users.role` — use DB lookup for RBAC) |
